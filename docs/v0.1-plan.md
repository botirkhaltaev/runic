## Crate Responsibilities

`ferralloc-core` owns allocator mechanics and must not depend on the public `ferralloc` crate, test support, or benchmarks.

`ferralloc` owns the public allocator type, global lock, and `GlobalAlloc` boundary.

`ferralloc-test-support` owns reusable test entities such as allocation traces, distributions, and subprocess abort cases.

`ferralloc-bench` owns benchmark workloads and reports.

## Core Invariant

```text
Every pointer returned by ferralloc belongs to exactly one span,
every span owns blocks of exactly one size class,
and every free must map back to a known span and block boundary.
```

## Architecture

```text
GlobalAlloc
  -> Allocator
      -> State
          -> SpanMap
          -> SpanTable
          -> Span
              -> FreeList
          -> OsMemory
```

Use one global lock around `State`.

## Entity Responsibilities

```text
Ferralloc    owns the Rust GlobalAlloc boundary
Allocator    owns the core public allocator API
State        owns allocation policy and the global lock-protected state
LayoutSpec   owns normalized layout semantics
SizeClasses  owns size-class selection
OsMemory     owns mmap and munmap
Span         owns small-block and large-allocation metadata
FreeList     owns the intrusive free-block chain
SpanTable    owns out-of-line span metadata storage
SpanMap      owns page-indexed pointer-to-span lookup
```

## Reference-Inspired Testing

Use `allocator-refs/` as read-only inspiration:

- Ferroc: randomized allocation traces and fuzz-style action sequences.
- Talc: alignment stress tests and small allocator microbench patterns.
- TCMalloc: size-class and span invariant tests.
- mimalloc-bench: realistic workload distributions and security misuse cases.
- snmalloc: future contention and remote-free benchmarks.

## Test Tiers

Default tests should cover size classes, free lists, span boundaries, span map lookup, central allocation/free, `alloc_zeroed`, `realloc`, large allocation, and `Box`/`Vec`/`String` smoke cases.

Abort tests should run in subprocesses and cover unknown pointers, small interior pointers, and large interior pointers.

Randomized tests should use deterministic seeds, small-biased sizes, mixed alignments, cookie validation, `alloc`, `free`, `realloc`, and `alloc_zeroed` actions.
